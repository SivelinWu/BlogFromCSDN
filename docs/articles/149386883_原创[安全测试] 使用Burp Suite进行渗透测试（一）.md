## 一、Burp Suite
Burp Suite 是由 ​**​PortSwigger Ltd.​** ​ 开发的一款​**​集成式 Web 应用程序安全测试工具​** ​，广泛应用于渗透测试、漏洞扫描和安全评估领域。
主要模块包括：
**模块**| **功能**| **场景**  
---|---|---  
**​​Proxy（代理）​​**|  拦截并修改浏览器与服务器之间的 HTTP/HTTPS 流量，支持实时编辑请求参数、Cookie 等。| 抓包调试、请求篡改、HTTPS 解密（需安装 CA 证书）  
**Scanner（扫描器）​**|  自动化检测漏洞（如 SQL 注入、XSS、CSRF），仅专业版支持。| 快速漏洞扫描与风险评估  
**​Intruder（入侵者）**|  高度可配置的自动化攻击工具，支持四种攻击模式：  
• Sniper（单点爆破）  
• Cluster Bomb（全组合爆破）  
• Pitchfork（多组同步）  
• Battering Ram（统一替换）| 密码爆破、参数枚举、边界测试  
​**​Repeater（重放器）**|  手动修改并重复发送请求，观察响应差异。| SQL 注入/XSS 手工验证、权限绕过测试  
**​Spider（爬虫）​**|  自动爬取网站目录结构，发现隐藏接口和资源。| 网站内容枚举与站点地图构建  
**​Decoder（解码器）​**|  支持 Base64、URL、HEX 等编码格式的转换与解码。| 敏感数据解密、Payload 构造  
下面对Burp Suite的基本使用进行介绍。

## 三、配置代理
打开Burp Suite先进行基本的配置，Proxy-Proxy Setting，配置你想要的代理配置，默认是127.0.0.1,8080接口。
然后再浏览器的设置里找到代理设置，以chrome为例，其实chrome用的是系统的代理配置
![](/cdn/149386883/12518fb606f045f484bb11364a07f237.png)
打开代理功能开关，填入上面配置的代理服务器的ip和接口
![](/cdn/149386883/39da0614c2714d2da3dbf2c5b03eab0f.png)
然后安装证书，在浏览器里访问<http://127.0.0.1:8080/>，在右上角点击下载证书
![](/cdn/149386883/0deff0dd1a654b589e4462a7af9055dc.png)
双击下载的证书按提示安装即可。
现在你在浏览器中访问地址就都会被Burp Suite抓包了。在Target-site map下可以看到请求的树状结构图。
![](/cdn/149386883/f89d0072cea84f118237a54d92fc92fc.png)
这里有个点要注意，Burp Suite是有劫持功能的，如果开启了劫持，你的访问被劫持了需要点击forward才能访问，所以会表现为你的请求都会超时。如果不需要劫持并修改接口，可以先关闭劫持功能。
![](/cdn/149386883/a6ac3a3c05464e8c9916a04551b51567.png)

#### 4.1、主动扫描
在Target-Site map下可以按照需要选择想要扫描的域名或者域名下的子分支，右击-actively scan
![](/cdn/149386883/3f8c821912c34a98bd8989780db0cef8.png)
在Dashboard下就可以看到扫描任务进行中
![](/cdn/149386883/2753d776773845fcbf5fb0db5ecab75d.png)
扫描结束后有结果的汇总
![](/cdn/149386883/b72ddf955f764b36a0adc6745dc5c870.png)
#### 4.2、被动扫描
在Dashboard下可以看到两个被动扫描任务，可以手动开启或者关闭。被动扫描就是只扫描经过代理被抓包的这部分接口。
**crawl** 主要发现请求并建立Site map下的树状图，它是会模拟用户行为，遍历和递归页面上的链接的。
**audit** 则是对漏洞的检测。

## 五、Intruder入侵
Burp Intruder作为Burp Suite中一款功能极其强大的自动化测试工具，通常被系统安全渗透测试人员被使用在各种任务测试的场景中。
它的工作原理是：Intruder在原始请求数据的基础上，通过修改各种请求参数，以获取不同的请求应答。每一次请求中，Intruder通常会携带一个或多个有效攻击载荷（Payload),在不同的位置进行攻击重放，通过应答数据的比对分析来获得需要的特征数据。
下面开始实操，首先在proxy-HTTP history中选择某个要入侵的接口，右击-send to intruder
![](/cdn/149386883/861262f4e6914522b9f34dce547767ca.png)
然后切到Intruder，可以看到刚才发过总来的请求，选中请求中需要替换进行攻击的参数，点击add$按钮（被$包围的参数在后续测试中会被payload中的参数替换掉），然后在右侧payload中新增需要尝试攻击的参数值，最后点击start attack开始模拟攻击。
![](/cdn/149386883/7ce611404e2e42049315209a4779b221.png)
在结果窗口中可以看到根据payload发起的攻击请求和响应
![](/cdn/149386883/9ddfb0d0e4fc4fa6bc95ce94cd966e16.png)
