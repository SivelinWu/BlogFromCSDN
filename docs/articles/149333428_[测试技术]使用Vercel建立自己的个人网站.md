# [测试技术]使用Vercel建立自己的个人网站

很多同学知道，在github中建立一个和账号同名的项目，github能实现自动部署，免费托管，还会给你一个域名访问自己的静态页面站点。所以很多会薅羊毛的同学会借助这个功能建立一个简单自己的个人网站。
但github的这个pages功能也有一下几个限制：
  * 仓库容量 ≤ 1GB，月流量 ≤ 100GB
  * 国内访问可能不稳定
  * 每个账号只能创建一个这样的项目


今天就来给大家介绍一个更加强大的工具：Vercel。
#### 🚀 ​**​Vercel 是什么？​** ​
Vercel 是由 ​**​Next.js 团队开发​** ​的云端部署平台，专注于静态网站、Serverless 函数和全栈应用的托管服务。其核心理念是 ​**​“零配置部署”​** ​，通过自动化流程大幅简化开发者的运维负担。
#### ⭐ ​**​核心优势详解​** ​
##### 1\. ​**​极速部署与自动化流程​** ​
  * ​**​秒级部署​** ​：代码提交到 Git 仓库（如 GitHub）后，Vercel 自动识别框架（React/Vue/Next.js 等），5 秒内生成可访问的 URL，无需手动配置构建脚本。
  * ​**​Git 深度集成​** ​：支持 GitHub/GitLab/Bitbucket，提交代码即触发部署，每次 Pull Request 生成临时预览链接，方便团队测试。


##### 2\. ​**​全球 CDN 加速与性能优化​** ​
  * ​**​70+ 边缘节点​** ​：覆盖全球的 CDN 网络，静态资源（图片、CSS/JS）自动压缩、缓存，访问延迟降至毫秒级。
  * ​**​自动 HTTPS​** ​：免费为自定义域名签发并自动续期 SSL 证书，确保数据传输安全。


##### 3\. ​**​强大的 Serverless 能力​** ​
  * ​**​无服务器函数​** ​：在项目根目录创建 `/api` 文件夹，编写 Node.js/Python 函数即可处理 API 请求，按需扩展，无需管理服务器。
  * ​**​无缝集成第三方服务​** ​：支持 AWS、Stripe、Twilio 等，快速构建动态功能（如支付、通知）。


##### 4\. ​**​开发者友好与免费额度​** ​
  * ​**​零配置框架支持​** ​：自动识别 Next.js、React、Vue、Angular 等 35+ 主流框架，生成优化构建命令。
  * ​**​慷慨免费套餐​** ​： 
    * 每月 100 次部署、100GB 流量、100 个无服务器函数、100,000 次 API 请求。
    * 支持私有仓库、自定义域名和环境变量管理。


##### 5\. ​**​高级功能拓展​** ​
  * ​**​多环境管理​** ​：通过 CLI 区分开发/生产环境，支持预发布测试。
  * ​**​A/B 测试与实时分析​** ​：基于部署预览进行流量分割测试，结合真实用户数据优化性能。


最重要的是就我们普通的使用，它是**免费的** 。
既然它这么强大，那部署一个（或者多个）个人网站，还不是手拿把掐。
## 一、部署项目到Vercel
#### 1.1、首先把你的静态页面的项目上传到gihub进行版本管理
这不就不细展开了~略
#### 1.2、打开Vercel网站，并注册登录。
略+1
#### 1.3、导入github项目
![](/cdn/149333428/f661e570e30245a6b4c60799e44c266f.png)
#### 1.4、导入完成后会显示你github上的项目，选择你要部署的项目
![](/cdn/149333428/cf6f92c9abee4d78ba17a5532226a043.png)
#### 1.5、配置&部署
Vercel 会自动检测你的项目类型，给出默认的构建配置好。一般不用特别修改。
点击deploy，Vercel 自动开始构建&部署你的项目了。等部署完成后，Vercel就可以在主页看到你的项目了。并且它会生产一个访问URL（<https://your-project-name.vercel.app>），打开这个URL就能成功访问到你的项目了。
## 二、映射到自己的域名
#### 2.1、首先你得注册一个属于你的域名，没有的话赶紧注册一个吧，具体步骤略+1。
#### 2.2、在域名后台添加解析映射，这边以阿里云为例。
需要配置CNAME类型的映射，解析到上面vercel给你的URL
![](/cdn/149333428/6f05143d48f34b5dab8a04f181765ef0.png)
#### 2.3、在vercel项目中配置上你的域名映射。
在vercel中你的项目的setting下面，进入domain，点击add domian，配置上你的域名
![](/cdn/149333428/c60e5d90a57140bcb34377fb006b4505.png)
操作没问题的话，这时你就可以通过自己的域名访问上面部署的站点了（DNS未刷新的话可能还得等一等）。
理论上到这一步就大功告成了，但我要加一个BUT，因为上面提到的vercel在国内是被禁止访问的。如何让没有梯子的人也能访问到你的站点呢。请看下一步。
