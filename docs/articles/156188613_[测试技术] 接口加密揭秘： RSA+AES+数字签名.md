## 一、背景：为什么接口要加密？
生活场景比喻：
你要给银行寄信：“请转100元给我的朋友张三，卡号：123456”
但路上快递员偷看信件或修改信件内容怎么办？

真实需求 |  方法  
---|---  
信不能被偷看 |  加密  
信的内容不能被偷偷替换 |  防篡改  
要确认是你本人寄的 |  身份验证  

解决方案：RSA+AES+数字签名
## 二、什么是RSA+AES+数字签名
### 2.1 AES——对称加密

项目 |  内容  
---|---  
全称 |  Advanced Encryption Standard（AES 算法）  
类型 |  对称加密（一把密钥开锁）  
速度 |  极快（硬件加速 AES-NI）  
密钥长度 |  128 / 192 / 256 位  
用途 |  文件加密、VPN、HTTPS 数据流  

优点：快！ 致命问题：如何安全地把密钥给对方？
类比：你和朋友用同一把锁加密信件，但得先把钥匙快递给他 —— 快递途中被截怎么办？
### 2.2 RSA——非对称加密

项目 |  内容  
---|---  
全称 |  Rivest-Shamir-Adleman（RSA 算法）  
类型 |  非对称密钥加密（一把锁，一把钥匙开）  
速度 |  较慢（比 AES 慢 ~1000 倍，仅适合小数据）  
密钥长度 |  2048 / 3072 / 4096 位（推荐 2048+）  
用途 |  密钥交换、数字签名、SSL/TLS 握手、证书签名  

优点：无需提前共享密钥！
致命问题：太慢，不能加密大数据！
类比：公钥 = 挂在门外的锁（谁都能锁），私钥 = 你口袋里的钥匙（只有你能开）
你把挂锁（公钥）寄给朋友用来锁箱子，但只有你有钥匙（私钥）能打开 —— 快递途中被截也没事！
假如有1000个箱子？累死了
### 2.3 数字签名

项目 |  内容  
---|---  
全称 |  Digital Signature（数字签名）  
类型 |  非对称加密应用（私钥签名，公钥验证）  
速度 |  较慢（只签哈希，数据越大越不影响）  
密钥长度 |  依赖算法（RSA-2048、ECDSA-256 等）  
用途 |  软件签名、SSL 证书、区块链交易、代码防篡改  

优点：证明“你发” + “没被改”！
致命问题：只能签名，不能加密内容！
类比： 你写信后用私章（私钥）对信的亲笔摘要（哈希）盖印 → 寄出 收信人用公章模板（公钥）比对印章 → 验证真伪 收信人用亲笔摘要与信的内容比对→ 验证没被篡改
但信的内容？明文寄！谁都看得懂！
### 2.4 RSA + AES混合加密 + 数字签名

项目 |  内容  
---|---  
全称 |  Hybrid Encryption with Digital Signature  
类型 |  AES 加密数据，RSA 加密密钥（公钥）+签名（私钥签名，公钥验证）  
速度 |  极快（大数据用 AES，仅密钥 + 签名用 RSA）  
密钥长度 |  AES: 256 位 + RSA: 2048 位  
用途 |  HTTPS、PGP 邮件、Signal 协议、代码签名 + 加密传输  

优点：又快又安全 + 防篡改 ！
致命问题：实现最复杂（但标准库都做好了）
## 三、 生活场景中的RSA + AES混合加密 + 数字签名
以背景引入的生活场景为例解析：
你要给银行寄信：“请转100元给我的朋友张三，卡号：123456”
### 3.1 信件该怎么加密？
  1. 把信锁进保险箱，设置随机密码：X7K9P2M4...
  2. 把保险箱密码写在纸条上，放进小铁盒，用银行的锁锁住（只有银行有钥匙打开）
  3. 印章贴纸背面写上内容摘要贴到保险箱（防伪签名）
  4. 寄出三样东西：


├── 小铁盒
├── 保险箱
└── 印章贴纸
### 3.2 银行收到信怎么打开？
  1. 用钥匙打开小铁盒 → 得到保险箱密码 X7K9P2M4...
  2. 成功打开保险箱 → 看到信件
  3. 用印章照片比对印章贴纸，并核对贴纸背后的摘要与信件内容一致→ 确认信是你寄的并且没被篡改


上述加密寄信的的操作，完美对应标准RSA + AES混合加密+签名协议。以下是其技术实现流程：
### 3.3 技术实现与生活场景映射
#### 3.3.1准备阶段
双方拥有各自的 RSA 密钥对：
发送方：私钥_发送方（保密）、公钥_发送方（可公开） 接收方：私钥_接收方（保密）、公钥_接收方（可公开）
#### 3.3.2发送方操作（加密 + 签名）
  1. 生成随机 AES 密钥 K→ AES密钥 K
↳ 对应：设置保险箱随机密码 X7K9P2M4...
  2. 用 密钥K 加密发送内容 → AES加密Content ↳ 对应：把信锁进保险箱
  3. 用公钥_接收方 加密 K → RSA公钥加密AES密钥 Key
↳ 对应：把密码写在纸条，放进小铁盒，用银行的锁锁住
  4. 用私钥_发送方对密文哈希做数字签名 → RSA私钥签名 sign
↳ 对应：盖私章 + 写摘要贴到保险箱上
  5. 打包发送：{ Key, Content, Sign}


↳ 对应：寄出 小铁盒 + 保险箱 + 印章贴纸
#### 3.3.3 接收方操作（验证 + 解密）
  1. 用私钥_接收方解密 Key → RSA私钥解密AES密钥 K
↳ 对应：用钥匙打开小铁盒，取出密码
  2. 用公钥_发送方验证 Sign 是否匹配 Content → RSA公钥验签sign是否匹配 Content
↳ 对应：比对印章 + 核对摘要与信件内容一致
  3. 用 K 解密 Content→ AES解密content


↳ 对应：用密码打开保险箱，看到信

    /** 从 Base64 公钥字符串加载 RSA 公钥 */
    public static PublicKey loadPublicKey(String publicKey) {
        try {
            byte[] keyBytes = Base64.getDecoder().decode(publicKey);               // 解码 Base64
            X509EncodedKeySpec spec = new X509EncodedKeySpec(keyBytes);            // X509 格式
            KeyFactory kf = KeyFactory.getInstance("RSA");                        // RSA 工厂
            return kf.generatePublic(spec);
        } catch(Exception e) {
            e.printStackTrace();
        }
        return null;
    }
    
    /** 从 Base64 私钥字符串加载 RSA 私钥 */
    public static PrivateKey loadPrivateKey(String privateKey) {
        try {
            byte[] keyBytes = Base64.getDecoder().decode(privateKey);             // 解码 Base64
            PKCS8EncodedKeySpec spec = new PKCS8EncodedKeySpec(keyBytes);         // PKCS8 格式
            KeyFactory kf = KeyFactory.getInstance("RSA");
            return kf.generatePrivate(spec);
        } catch(Exception e) {
            e.printStackTrace();
        }
        return null;
    }
    
    /** 使用 RSA 公钥加密明文，返回 Base64 密文 */
    public static String encryptRSA(String plainText, PublicKey key) {
        if (key == null) return "";
        try {
            Cipher cipher = Cipher.getInstance("RSA/ECB/PKCS1Padding");           // RSA 加密模式
            cipher.init(Cipher.ENCRYPT_MODE, key);
            return Base64.getEncoder().encodeToString(
                    cipher.doFinal(plainText.getBytes(StandardCharsets.UTF_8)));
        } catch(Exception e) {
            e.printStackTrace();
        }
        return null;
    }
    
    /** 使用 AES 密钥加密内容，返回 Base64 密文 */
    public static String encryptAES(String content, String key) {
        if (key == null) return null;
        try {
            SecretKeySpec aesKey = new SecretKeySpec(key.getBytes(StandardCharsets.UTF_8), "AES");
            Cipher aesCipher = Cipher.getInstance("AES/ECB/PKCS5Padding");        // AES 加密模式
            aesCipher.init(Cipher.ENCRYPT_MODE, aesKey);
            byte[] encryptAESed = aesCipher.doFinal(content.getBytes(StandardCharsets.UTF_8));
            return Base64.getEncoder().encodeToString(encryptAESed);
        } catch(Exception e) {
            e.printStackTrace();
        }
        return null;
    }
    
    /** 生成指定长度的随机字符串（字母+数字） */
    public static String generateRandomString(int length) {
        String chars = "ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789";
        Random random = new Random();
        StringBuilder sb = new StringBuilder(length);
        for (int i = 0; i < length; i++) {
            sb.append(chars.charAt(random.nextInt(chars.length())));
        }
        return sb.toString();
    }
    
    /** PKCS7 填充，使字符串长度为 16 的倍数 */
    public static String padString(String input) {
        int blockSize = 16;
        int padLength = blockSize - (input.length() % blockSize);
        StringBuilder padded = new StringBuilder(input);
        for (int i = 0; i < padLength; i++) {
            padded.append((char) padLength);
        }
        return padded.toString();
    }
    
    /** 使用 RSA 私钥对数据进行 SHA1withRSA 签名，返回 Base64 */
    public static String signData(String data, PrivateKey key) {
        if (key == null) return null;
        try {
            Signature signer = Signature.getInstance("SHA1withRSA");              // 签名算法
            signer.initSign(key);
            signer.update(data.getBytes(StandardCharsets.UTF_8));
            return Base64.getEncoder().encodeToString(signer.sign());
        } catch(Exception e) {
            e.printStackTrace();
        }
        return null;
    }

            return new String(decryptedBytes, StandardCharsets.UTF_8);
        } catch(Exception e) {
            log.error("RSA decrypt failed: " + e.getMessage(), e);
            return null;

            return false;

### 4.3 代码加密解密流程解析
#### 4.3.1核心组件功能对照表

组件 |  作用 |  核心原理 |  对应代码 |  输入 → 输出  
---|---|---|---|---  
AES |  加密Content |  随机生成密钥 K 加密内容 |  `encryptAES()` |  明文 → 密文  
解密Content |  使用密钥 K 解密密文 |  `decryptAES()` |  密文 → 明文  
RSA |  加密key |  用公钥加密 密钥K |  `encryptRSA()` |  K → 加密 K  
解密key |  用私钥解密得到原始密钥K |  `dencryptRSA()` |  加密 K → K  
数字签名 |  加密签名Sign |  私钥签名 |  `signData()` |  数据 → 签名  
验证签名Verify |  公钥验证签名是否匹配原数据 |  `verifyData()` |  数据 + 签名 → 验证  

#### 4.3.2 请求加密 & 参数组装（发送前处理）
把原始明文请求 → 加密 + 签名 → 替换成密文参数，发送给支付平台。

#### 4.3.4 响应解密 & 断言校验（接收后处理）
接收加密响应 → 验签 → 解密 → 校验业务字段是否正确。
// 核心流程
boolean signValid = verifySign(encryptedContent, encryptedSign, 平台公钥);// 用平台公钥RSA验签
String randomValue = decryptRSA(encryptedKey, 商户私钥);// 用商户私钥 RSA 解密AES密钥
String decryptedContent = decryptAES(encryptedContent, randomValue);// 用AES密钥解密响应体
JSONObject jsonObject = JSON.parseObject(decryptedContent);//解析json
boolean hasOrderNo = jsonObject.containsKey("orderNo");//断言字段
